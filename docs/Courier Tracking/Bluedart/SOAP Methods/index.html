SOAP based methods Overview

Info

For detailed instructions refer to original documentation.

Finders#

Pincode#

To find the services available by bluedart for a particular pincode and product.

WSDL : https://netconnect.bluedart.com/Ver1.9/ShippingAPI/Finder/ServiceFinderQuery.svc?wsdl

Methods used in location finder:

1. GetServicesforPincode( pinCode, profile )#

All type of services available for particular pincode.

2. GetServicesforProduct( pinCode, pProductCode, pSubProductCode, profile)#

Get Service availability for particular pincode, product and subproduct.

3. GetServicesforPincodeAndProduct( pinCode, ProductCode, SubProductCode, PackType, Feature, profile )#

Get Service availability for particular pincode, product and subproduct.

Transit Time#

Provides you with the delivery date for a selected service.

WSDL : https://netconnect.bluedart.com/Ver1.9/ShippingAPI/Finder/ServiceFinderQuery.svc?wsdl

Method used:

GetDomesticTransitTimeForPinCodeandProduct( pPinCodeFrom, pPinCodeTo, pProductCode, pSubProductCode, pPudate, pPickupTime, profile )#

Expected delivery date of the shipment when it is shipped on a certain date.

Pickup#

Pickup Registeration#

The Pickup registration API exposes methods those will be used to register a Pickup.

WSDL : https://netconnect.bluedart.com/Ver1.9/ShippingAPI/Pickup/PickupRegistrationService.svc?wsdl

Methods used:

1. RegisterPickup( PickupRegistrationRequest, Profile )#

Pickup registration.

2. GetAllProductsAndSubProducts( Profile )#

This method will return all product & subproduct codes details that will be used to send as input parameters in Pickup Registration method.

Pickup Cancel#

WSDL : https://netconnect.bluedart.com/Ver1.9/ShippingAPI/Pickup/PickupRegistrationService.svc?wsdl

Allow user to cancel pickup (which are registered by him).

Method:

CancelPickup( Request, Profile )#

WayBill#

WSDL : https://netconnect.bluedart.com/Ver1.9/ShippingAPI/WayBill/WayBillGeneration.svc?wsdl

WayBill Generation#

Methods used:

1. GenerateWayBill( WayBillGenerationRequest, Profile )#

To generate way bill.

2. ImportData( List<WayBillGenerationRequest>, Profile )#

To generate multiple way bill at once.

3. CancelWaybill( AWBCancelationRequest, Profile )#

Cancels waybill before shipment manifest.

4. UpdateEwayBill( EwayBillRequest, Profile )#

Provision to update Ewaybill number generated by customer against blue dart waybill numbers.

Pincode Master Download#

WSDL : https://netconnect.bluedart.com/Ver1.9/ShippingAPI/Master/MasterDownloadQuery.svc?wsdl

Method:

DownloadPinCodeMaster( lastSynchDate, Profile )#

Will downloads incremental pin code master changes.

ALTInstruction Update#

Used to update Alternate instructions for a Waybill number.

WSDL: https://netconnect.bluedart.com/Ver1.9/ShippingAPI/ALTInstruction/ALTInstructionUpdate.svc?wsdl

CustALTInstructionUpdate( ALTInstructionRequest, Profile )#

This method will also work with REST by using the following url and POST request.

http://netconnect.bluedart.com/Ver1.9/Demo/ShippingAPI/ALTInstruction/ALTInstructionUpdate.svc/rest/CustALTInstructionUpdate

Sample Request Data for RTO:

{"altreq": {"AWBNo": "69501388751","PreferDate": "/Date(1517893738847)/","AltInstRequestType": "RTO","TimeSlot": "","MobileNo": "","PreferTime":""},"profile": {"LoginID": "Valid Login ID","LicenceKey": "Valid Key","Api_type": "S","Version": "1.9"}}

But the reponse will be XML based. Sample Response of DT :

<Responsexmlns="http://netconnect.bluedart.com/Ver1.9/Demo/ShippingAPI/ALTInstruction/ALTInstructionUpdate.svc/rest/CustALTInstructionUpdate"><CustALTInstructionUpdateResult><AWBNo>59500670843</AWBNo><IsError>false</IsError><status><e><StatusCode>Valid</StatusCode><StatusInformation>AltInstrucation Update Successful</StatusInformation></e></status></CustALTInstructionUpdateResult></Response>
