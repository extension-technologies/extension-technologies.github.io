nodeProvisioning

Galera Article: https://galeracluster.com/library/documentation/node-provisioning.html

Variable view: SHOW VARIABLES LIKE '%wsrep_cluster_address%'; SHOW GLOBAL VARIABLES LIKE '%wsrep_cluster_address%'; SHOW STATUS LIKE 'wsrep_cluster_size';

Set Variable: SET GLOBAL max_error_count=128;

TO ADD A NEW NODE TO CLUSTER Link: https://www.globo.tech/learning-center/how-to-add-a-new-node-to-a-galera-replication-cluster-on-ubuntu-14-04-lts/

  2. Make sure you are able to connect with those databases with mysql -u root -ppasswwd -h 10.122.0.24

  3. Copy the /etc/mysql/my.cnf and /etc/mysql/debian.cnf files to current server.

TO REMOVE A NODE FROM CLUSTER Temp: Just stop the service of that database on the node we are going to remove till we can restart the infra. Permanent:

  1. Stop the service of the database on node we are going to remove.
  2. Now first start restarting the child nodes by changing the mysql.cnf file then at last the master database.
